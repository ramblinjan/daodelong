
So, I am working on an initial prototype of a system -- v1 will be a federated graphql API that has a few basic functions, but we will effectively have a self-editing LLM-powered AI system. We will use graphql features like introspection for the system itself as it is its own developer. Our first step, though, is to write this core loop where it is prompted to make a decision about a change and it can call against its own endpoint to create, edit its own code. As such, part of the constraint of the system is no "build" step that goes from readable code to executed -- we need the code to be constantly living. Here is some of the essence in the readme:
